Pipeline Phase Guide

How This Project Works

This application uses a deterministic, rule-based compilation pipeline to convert plain English commands into Python code. Each phase has a single responsibility: tokenization, structure validation, meaning checks, intent extraction, intermediate representation, and final code generation.

Phase Examples

Phase 1

Lexical Analysis

Input: add 4 and 5
Output: KEYWORD(add), NUMBER(4), WORD(and), NUMBER(5)

Phase 2

Syntax Analysis

Input Tokens: KEYWORD NUMBER WORD NUMBER
Output: matches rule ADD_NUM_NUM

Phase 3

Semantic Analysis

Input: rule + tokens
Output: valid = true, message = semantic checks passed

Phase 4

Intent and Entities

Input: add 4 and 5
Output: intent = add, entities = [4, 5]

Phase 5

Intermediate Representation

Input: intent and entities
Output: { op: "add", args: [4, 5] }

Phase 6

Code Generation

Input: IR add operation
Output: result = 4 + 5
